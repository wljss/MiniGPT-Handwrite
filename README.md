一些问答

### 为什么要除以 √d_k？

**防止点积数值过大导致 Softmax 梯度消失，保证训练稳定性。**

两个均值为0、方差为1的随机向量（假设为d_k维）做内积，结果的方差等于向量维度 `d_k`（每个头的隐藏维度）。当 `d_k` 很大时，点积的数值会非常大。如果点积数值过大，Softmax 函数的输出会被“拉向”极端值（大部分接近0，少数接近1）。此时 Softmax 的梯度会趋近于0，导致反向传播时梯度消失，模型无法训练。除以 `√d_k` 可以将点积的方差归一化到1，让 Softmax 的输入保持在合理的数值范围，梯度能够正常流动。

### 为什么多头比单头强？

**多头能让模型同时学习多种不同的注意力模式，在不显著增加参数的前提下大幅提升表达能力。**

单头注意力只能学习一种全局的语义关系。比如它要么关注“语法相邻关系”，要么关注“语义相似关系”，无法同时捕捉多种不同的依赖。多模式并行学习，每个头独立学习不同的注意力模式。比如有的头学习主谓宾语法关系，有的头学习指代关系，有的头学习局部相邻关系，有的头学习长距离依赖。最后通过 `Wo` 矩阵将多个头的输出融合，得到综合了所有注意力模式的最终表示。

### 为什么 GPT 不需要 Encoder？
**GPT 的核心任务是自回归语言建模（根据前文预测下一个token），只需要单向上下文信息，纯 Decoder 架构已经足够。**

- **Transformer 三种架构的分工**：
  - **Encoder-only（BERT）**：双向注意力，能看到所有位置的token，适合理解类任务（分类、实体识别）。
  - **Encoder-Decoder（T5、翻译模型）**：Encoder 编码源文本（双向），Decoder 生成目标文本（单向），适合需要“输入-输出”映射的任务（翻译、摘要）。
  - **Decoder-only（GPT）**：带 Causal Mask 的单向注意力，只能看到当前及之前的token，完美匹配自回归生成任务。
- **Encoder 对 GPT 是多余甚至有害的**：
  - Encoder 的双向注意力会让模型在预测下一个token时“看到未来的token”，这属于作弊，会导致模型无法学习到正确的生成逻辑。
  - GPT 的任务不需要额外的“源输入编码”过程，它只需要基于已生成的文本继续生成，因此不需要独立的 Encoder 模块。

### 为什么 KV Cache 能加速推理？
**通过缓存历史计算结果，将每次推理的时间复杂度从 O(T²) 降低到 O(T)，彻底解决“越生成越慢”的问题。**

`generate` 函数每次生成新token时，都会把整个 `input_ids` 输入模型，重新计算所有位置的 Q/K/V 和注意力。比如生成第10个token时，要重新计算前9个token的所有计算结果，这是完全重复的浪费。
- **KV Cache 的原理**：
  1. 第一次推理时，计算所有位置的 K 和 V，并缓存起来。
  2. 后续每次生成新token时，**只计算新token的 Q/K/V**，不需要重新计算历史token的 K/V。
  3. 将新的 K/V 拼接到缓存中，用完整的 K/V 和新的 Q 计算注意力。
- **性能提升**：
  - 时间复杂度：从 O(T²) → O(T)，生成1000个token的速度比原生快几百倍。
  - 代价：需要额外的显存存储 KV Cache，空间复杂度为 O(L*H*T*Dh)（L是层数，T是序列长度）。

### 为什么长上下文难？
**长上下文在计算、内存、模型能力三个维度都存在根本性挑战。**

1. **计算复杂度爆炸**：
   自注意力的核心运算 `Q·K^T` 的时间复杂度是 O(T²)。当 T=1024 时，运算量是100万次；当 T=128k 时，运算量是160亿次，普通GPU根本无法承受。

2. **内存占用剧增**：
   KV Cache 的内存占用随序列长度线性增长。以7B模型为例，128k上下文的KV Cache会占用约40GB显存，远超消费级GPU的显存容量。

3. **注意力稀释问题**：
   Softmax 的输出总和为1。当序列很长时，注意力权重会被大量token“稀释”，模型很难将注意力集中在少数关键的远距离token上，导致长距离依赖丢失。

4. **位置编码外推性差**：
   即使是RoPE，当序列长度远超训练时的最大长度时，旋转角度的分布会偏离训练分布，导致模型对相对位置的判断出现偏差，生成质量急剧下降（这也是Yarn等RoPE改进方案要解决的问题）。

5. **训练数据偏差**：
   大部分预训练数据都是短文本（平均长度几百token），模型在训练过程中很少见到长文本，因此天生缺乏处理长上下文的能力。
